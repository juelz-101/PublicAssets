# AI Rules for i18n Module

This module provides a simple internationalization service for React applications.

## Usage Guidelines for AI

1.  **Initialization**: Always call `createI18n` at the root of the application (e.g., in `main.tsx` or `App.tsx`) before using the hook.
    ```typescript
    createI18n({
      translations: { en: { hello: 'Hello' }, es: { hello: 'Hola' } },
      defaultLocale: 'en'
    });
    ```

2.  **Translation Hook**: Use `useTranslation()` within functional components to access the `t` function.
    ```typescript
    const { t, locale } = useTranslation();
    return <h1>{t('hello')}</h1>;
    ```

3.  **Interpolation**: The `t` function supports interpolation using `{key}` syntax in translation strings.
    - Translation: `welcome: "Welcome, {name}!"`
    - Usage: `t('welcome', { name: 'User' })`

4.  **Locale Switching**: Use `setLocale(newLocale)` to change the language globally.

## Constraints
- Do not attempt to use `t` outside of a React component or without the hook unless using the raw `t` export (which is not reactive).
- Ensure all translation keys are present in all supported locales to avoid missing key warnings.
