Ok I've made some crutial changes to the structure of the manifest and the data for the sections.
Heres the list of the changes made, and why I made them:

Manifest:
	- Root Structure: Restructured the root keys to match the rest of our projects manifests that use our db.ts module.
		^ The main change being that all of the data-based arrays are now contained within data/ instead of all being in the root.
	- Player: Added settings for the playback of music, and added default cfg parameters to be used when the item doesnt have the cfg key set.
	- Flags: Added Global flags to be assigned to items, all categorised for different purposes: 
		1. Special: Flags to assign items to other sections (ie: featured, new releases)
		2. Status: Flags to indicate the current status of the item.
		3. System: Flags to tell the system how the item should be handled.
		!. Edit: Ive added some more, they are self-explanitory.
		^. The targets key is to set where the flag can be used. If target key is missing then it can be used anywhere.
	- Auto-Populate: Added a toggle to auto-populate certain sections during inital loading of the data (ie: Albums tracklists and artists should auto-populate based on the songs data, etc. Items can still be added manually (ie: Assign tracks to albums before the track has been made), with the "overwrite_existing_items" boolean added to set how conflicts with existing items that were added manually are handled. 
	
Music.json: (Will apply to other sections aswell)
	- Replaced "debug", "featured" and "new_release" keys with the "flags" array.
	- Removed data that is auto-populated, but left an example of an item being added manually (assigned a future song to the track number)
	
art.json:
	- Moved all art arrays into a new array called "galleries" for a cleaner json layout, the UI in the app is to remain how it is now, we only need to add the parent "galleries" array to the paths to load.
	- Changed the "medium" key within items to "category", as the medium is defined by the parent arrays key.
	- Added the auto-populate arrays to the artists.
	- Added the flags to all items.
	
gaming.json:
	- Added developers to root and to each item ready to auto-populate.
	- Added the flags to all items.
	
tools.json:
	- Added developers to root and to each item ready to auto-populate.
	- Added the flags to all items
	
community.json:
	- Split the "people" array into people.organisers, people.volunteers and people.guests.
	- Changed "organised_by" in each event to "people" ready to auto-populate the main people arrays.
	- Added the flags to all items.
