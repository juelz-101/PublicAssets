## Page 1: The "AI-First" IDEs (Project Context)

If you want the AI to understand your entire project folder—not just the file you're looking at—you need an editor that indexes your codebase.

* **Cursor:** * **The Good:** Still the benchmark for "codebase-aware" editing. It indexes your project and uses `@-mentions` to reference specific files or docs.
    * **The Bad:** The free tier is "restrictive" compared to the paid Ultra plan. 
* **Trae:** * **The Good:** A ByteDance-backed alternative that is currently "fully free" to lure users away from Cursor. It matches the speed and multi-file editing capabilities of paid tools.
* **Windsurf:** * **The Good:** Uses the "Cascade" agent which is more "agentic"—it watches your terminal and file changes more passively to build context.
    * **The Bad:** The free plan is limited to 25 prompts per month. That’s about twenty minutes of work for someone who doesn't know what they're doing.

## Page 2: The "Independence" Stack (Extensions & APIs)

If you want to stop being a victim of "UI nerfs," use an open-source extension and plug in your own model.

* **Continue.dev:** * An open-source extension for VS Code and JetBrains. It allows you to swap models (like Claude, GPT-4, or local models via Ollama) so you aren't locked into one provider’s mood swings.
* **GitHub Copilot (Free Tier):** * As of early 2026, GitHub offers a free tier with 2,000 code completions and 50 chat messages per month. 
    * **Warning:** It does *not* include the high-end "agent mode" or multi-file editing on the free plan. It’s a tool for light assistance, not project rebuilding.
* **Cline (formerly Claude Dev):** * An open-source agent that lives in your sidebar. It can run terminal commands and edit files autonomously if you give it an API key.

## Page 3: Workflow, Files, and Issue Tracking

Since you’re "done with Google," you need to manage your projects like a professional. 

* **Linear:** * The "lightning-fast" alternative to Jira. It’s keyboard-driven and keeps your tasks organized without the bloat. The free tier is generous for solo developers.
* **Jira:** * Still the industry standard for agile development. Free for up to 10 users. If you want to work in a real company, you should probably learn how to use this monstrosity.
* **MkDocs & GitHub:** * Use **MkDocs** to build your own static documentation sites from Markdown—no cloud subscription required.
    * **GitHub** remains the mandatory choice for version control and collaboration.

## Page 4: Learning Resources: The "Use Your Brain" Section

You said you wanted to learn. Here is where the real work starts. If I catch you copy-pasting from a chatbot without understanding the syntax, I'm failing you.

* **TypeScript Official Handbook:** * The only authoritative source for the language. It covers everything from basic types to complex generics and "Mapped Types".
* **Scrimba (Learn TypeScript):** * A 4.2-hour interactive course where you pause the video and code directly in the player. Highly recommended for people who have the attention span of a goldfish.
* **Total TypeScript (Matt Pocock):** * While many parts are paid, his free tutorials are the gold standard for intermediate-to-advanced patterns.
* **DevDocs.io:** * An essential tool that aggregates documentation for almost every language and framework (TypeScript, React, Node, etc.) into a single, searchable, offline-capable interface.

## Page 5: Finishing Page

Google AI Studio was a "pot of gold" that everyone knew would eventually run dry. Moving to **Cursor** or **Trae** will give you back your multi-file context, but the real solution is learning **TypeScript** properly through the **Official Handbook** and **Scrimba**. 

If you get stuck, don't just ask the AI to fix it—ask it to *explain* why your current approach is garbage. That’s how you actually get better.
