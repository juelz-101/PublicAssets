## Page 1: Introduction

**Google AI Studio App Builder** is a specialized tool within Google AI Studio (`aistudio.google.com/apps`) that allows you to generate and deploy full-stack web applications from a simple natural language prompt. It provides a seamless transition from idea to a working, editable codebase, making it a powerful resource for rapid prototyping and development.

## Page 2: What is Google AI Studio App Builder?

  - A web-based development environment that uses Gemini to generate a complete application from a text prompt.
  - It moves beyond simple text generation to create a fully-fledged, runnable codebase.
  - The generated apps are typically built on modern web frameworks, including **TypeScript**, React, and Vite.
  - You can preview the app, download the source code, and deploy it directly to Google Cloud.
  - **Not just a playground:** It's designed to create a functional MVP (Minimum Viable Product) that can be extended and customized.

## Page 3: Key Features & Core Technologies

| Feature | Description | Core Technologies |
|---|---|---|
| **Code Generation** | Creates a complete, well-structured web application from a single prompt. | Gemini API, React, Vite |

## Page 4: Getting Started

1.  **Go to the Google AI Studio App Builder:** Navigate directly to `https://aistudio.google.com/apps`.
2.  **Sign in with your Google Account:** Access the tool with your existing Google account.
3.  **Provide a Prompt:** Describe the app you want to build in a single, clear sentence.
4.  **Generate and View:** The AI will begin generating the code and will present a live preview of the app on the right-hand side.
5.  **Explore and Refine:** Use the integrated IDE to inspect the generated files and make adjustments.

## Page 5: Prompting for App Generation

When using the App Builder, your prompt acts as a blueprint. A well-crafted prompt is crucial for generating a functional and relevant application.

**Example Prompts:**

  - **Simple:** `Generate a web app that takes a user's name and favorite color and displays a welcome message in that color.`
  - **Complex:** `Create a full-stack web app with a form that accepts a video URL. The app should use FFMPEG to extract a 10-second clip from the video, send it to the Gemini API with a prompt to summarize the action, and then display the summary and the clip.`
  - **Multimodal:** `Build an app that lets a user upload an image of a dish and get a recipe and a nutritional summary from the Gemini API.`

-----

## Page 6: Prompt Engineering Best Practices

  - **Be Specific:** Don't just say "make a chatbot." Instead, say "make a single-page chatbot app with a sleek, dark-themed UI that answers questions about space exploration."
  - **Specify Technologies:** If you have a preference, include it. "Use React and Tailwind CSS to build..."
  - **Define Inputs and Outputs:** Clearly state what information the user provides and what the app should produce in response.
  - **Break down complex tasks:** While the AI can handle complex requests, it's often better to start with a simpler app and then add more features by refining the code.

## Page 7: Exporting and Running Locally

Once you have generated your app, you can download the source code as a ZIP file. The generated project is a standard Node.js application.

### Basic Instructions to Run After Export

1.  **Extract the ZIP file:** Unzip the downloaded project to a directory on your computer.
2.  **Open a Terminal:** Navigate to the project's root directory in your terminal or command prompt.
3.  **Install Dependencies:** Run the following command to install all required packages:
    ```bash
    npm install
    ```
4.  **Set Up API Key:** The Gemini API key should be stored in a `.env` file for security. Create a file named `.env` in the root of your project and add your key:
    ```
    GEMINI_API_KEY=your_api_key_here
    ```
5.  **Run Development Server:** Start the local development server with the following command:
    ```bash
    npm run dev
    ```
    Your application should now be running in your browser, typically at `http://localhost:5173`.

## Page 8: Setting Up External Dependencies

The generated apps often use a boilerplate that can be extended with external tools.

### Tailwind CSS Setup

Generated apps use CDN to import Tailwind, this may lead to security errors especially when using other external dependencies.
It is recommended to install Tailwind locally and let Vite build it instead. This can be done by following these steps:

1.  **Install Tailwind Dependencies:**
	Open your terminal to your project root and put the following:
    	```bash
    	npm i -D tailwindcss @tailwindcss/vite
    	```
2.  **Create styles.css:**
	Create a new file called styles.css (we will add it to the project root for simplicity)
	Open the file and add the following:
    	```css
	@import "tailwindcss";
    	```

3.  **Link styles.css:**
	Import the new script in your app entry (usually app.tsx or similar).
	In this file you should see a list of imports near the top of the script, add a new line with the following:
    	```ts
    	import './styles.css'
    	```

4. **Setup vite.config.ts:**
	From the project root open `vite.config.ts` and add the following line to the top of the script, under the last `import`:
	```ts
	import tailwindcss from '@tailwindcss/vite'
	```

	Just below, look for where it says `return` and add `tailwindcss()` as a `plugin`:
	```ts
	const env = loadEnv(mode, '.', '')
  	return {
    		plugins: [tailwindcss()],	// Add this.
    		define: {...},
	},
	```

5.  **Remove CDN Import:** 
	Open index.html and remove the following line
    	```html
    	<script src="https://cdn.tailwindcss.com"></script>
    	```

Note: No `tailwind.config.js` is required for basic use in v4+, although you may add one if you'd like to add custom themes.
	Tailwind Custom Themes Setup: `../css/@id:css_tw_ct`

A zip file containing these instructions with a ready-made styles.css and vite.config.ts to be dragged into your project can be downloaded here:
	Tailwind Setup Bin: `../css/res/zi_tailwind_setup.zip`


### FFMPEG Setup for Video/Audio Processing

For applications that require multimedia processing, you will need to set up FFMPEG.

1. **Install FFMPEG Dependencies:**
	Open your terminal to your project root and put the following:
    	```bash
	npm i @ffmpeg/core
	```

2. **Configure Headers:**
	The following headers are required to enable SharedArrayBuffer in modern browsers, which FFmpeg WebAssembly needs for high-performance operations.
	From the project root, open the file `vite.config.ts`
	Edit to the `server` and `preview` headers to look like this:
	```ts
	optimizeDeps: {
		exclude: ['@ffmpeg/ffmpeg', '@ffmpeg/utils']
	},
	server: {
      		// Needed for SharedArrayBuffer (cross-origin isolation)
      		headers: {
        		'Cross-Origin-Opener-Policy': 'same-origin',
        		'Cross-Origin-Embedder-Policy': 'require-corp',
        		// optional helpers while developing
        		'Cross-Origin-Resource-Policy': 'cross-origin',
        		'Access-Control-Allow-Origin': '*',
      		},
    	},
    	preview: {
      		headers: {
        		'Cross-Origin-Opener-Policy': 'same-origin',
        		'Cross-Origin-Embedder-Policy': 'require-corp',
        		'Cross-Origin-Resource-Policy': 'cross-origin',
        		'Access-Control-Allow-Origin': '*',
      		},
    	},
	```
  - **Local Development:** The instructions above are for development on a local system.
	A zip file containing these instructions with a ready-made vite.config.ts to be dragged into your project can be downloaded here:
	FFMPEG Setup Bin: `../vite/res/zi_vite_ffmpeg_setup.zip`

	**Imports:**
	import { AppState, ExportOptions, ExportProgress } from '../types';
import { FFmpeg } from '@ffmpeg/ffmpeg';
import { fetchFile } from '@ffmpeg/util';
import coreURL from "@ffmpeg/core?url";
import wasmURL from "@ffmpeg/core/wasm?url";

  - **Deployment:** For a production environment like Cloud Run, you must ensure FFMPEG is available in your container. This is done via the `Dockerfile`. An example `Dockerfile` snippet to install FFMPEG would look like this:

    ```dockerfile
    FROM node:18-slim
    USER root
    RUN apt-get update && apt-get install -y ffmpeg
    ...
    # Your app's code and setup
    ```

    This adds the CLI version of FFMPEG to the container's environment, allowing your deployed application to use it for tasks like video clipping or format conversion.

## Page 12: Deployment and Production

Google AI Studio offers a seamless deployment option, but it's important to understand the underlying process.

### One-Click Deployment

  - Click the "Deploy to Cloud Run" button in the App Builder.
  - The tool will automatically containerize your application, upload it to a container registry, and deploy it to a scalable, serverless environment on Google Cloud Run.
  - **Important:** This process requires a Google Cloud project with billing enabled.

### Setting Up Dependencies for Deployment

For production, security and performance are key.

  - **API Keys:** Never hardcode your API key. Always use environment variables. Google Cloud Run provides a secure way to manage these in the console.
  - **Required APIs:** Ensure your Google Cloud project has the necessary APIs enabled, such as `Cloud Run Admin API`, `Cloud Build API`, and `Artifact Registry API`. The App Builder will often guide you to enable these.
