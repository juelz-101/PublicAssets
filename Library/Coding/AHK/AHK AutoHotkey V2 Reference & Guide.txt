```markdown
## Page 1: Introduction

**AutoHotkey V2** (AHK V2) is a powerful, free scripting language for Windows automation.  
It allows you to automate repetitive tasks, create hotkeys, manipulate windows, and much more with concise, readable scripts.

- **Organisation:** ZIKYinc  
- **Author:** Mojula Patel  
- **Approved by:** Juelz-101

---

## Page 2: What is AHK V2?

- AHK V2 is the latest major version of AutoHotkey.
- It features a modernized, consistent syntax and improved reliability.
- **Not backward-compatible with V1:** Scripts written for V1 will often need changes to run on V2.
- Used for:  
  - Keyboard/mouse automation  
  - GUI scripting  
  - Macro creation  
  - Windows and process management

---

## Page 3: Key Differences (V1 vs V2)

| Feature             | AHK V1                        | AHK V2 (Current)              |
|---------------------|------------------------------|-------------------------------|
| Assignment          | `=` (legacy), `:=` (expr.)   | Only `:=`                     |
| Function Calls      | `Func, Param1, Param2`       | `Func(Param1, Param2)`        |
| Hotkey Syntax       | `::` and legacy forms        | Only modern syntax            |
| Variables           | `%var%` for legacy expansion | `%` not needed, direct use    |
| Objects/Arrays      | Limited, inconsistent        | Unified, modern objects/arrays|
| Error Handling      | `If ErrorLevel`              | Try/Catch, Exceptions         |
| GUI                 | Legacy commands              | Object-oriented GUI           |

## Page 4: Basic Syntax

```
; Single-line comment
MsgBox "Hello, world!"  ; Inline comment

; Variables
name := "Mojula"
count := 5

; Expressions
result := count * 2
```

- All assignments use `:=`
- Strings are in double quotes
- Comments use `;`

---

## Page 5: Variables, Expressions, and Types

- **Variables**: No need for `%` expansion.
- **Types**: String, Integer, Float, Object, Array, Boolean.
- **Assignment**: Always use `:=`

```
x := 10
y := "Hello"
arr :=
obj := {name: "ZIKYinc", year: 2025}
```

---

## Page 6: Hotkeys and Hotstrings

**Hotkeys:**  
```
^j::MsgBox "Ctrl+J was pressed!"    ; ^ = Ctrl
#n::MsgBox "Win+N was pressed!"     ; # = Win key
```

**Hotstrings:**  
```
::btw::by the way
::addr::123 Main Street, London
```

- Hotkeys trigger actions on keypress.
- Hotstrings expand text as you type.

---

## Page 7: Functions and Methods

**Function Definition:**
```
add(x, y) {
    return x + y
}

MsgBox add(2, 3)
```

**Methods and Objects:**
```
person := {name: "Mo", greet: greet}
greet(this) {
    MsgBox "Hello, " this.name "!"
}
person.greet()
```

- Functions use parentheses for parameters.
- Objects and methods are first-class.

## Page 8: GUI and Windows Automation

**Simple GUI:**
```
myGui := Gui()
myGui.Add("Text",, "Enter your name:")
myGui.Add("Edit", "vName")
myGui.Add("Button", "Default", "OK").OnEvent("Click", (*) => MsgBox "Hello, " myGui["Name"].Value)
myGui.Show()
```

**Window Automation:**
```
WinActivate "Untitled - Notepad"
Send "Hello, Notepad!"
```

- GUIs are object-based.
- Window functions are modernized.

## Page 9: Libraries and Includes

- Use `#Include` to add external scripts or libraries.
- Many community libraries are available for AHK V2.

```
#Include MyLibrary.ahk
```

- Functions and classes from included files become available.

## Page 10: Best Practices

- **Always use `:=` for assignment.**
- **Use functions and objects** for modular code.
- **Comment your scripts** for clarity.
- **Handle errors** with `try/catch`.
- **Prefer meaningful variable and function names.**
- **Test scripts in small pieces** before combining.

## Page 11: Usage Examples

**Example 1: Simple Hotkey**

```
^s::MsgBox "You pressed Ctrl+S!"
```

**Example 2: Loop and Array**

```
arr := [1, 2, 3, 4,alue in arr

```
**Example 3: Try/Catch**

```

    FileDelete "C:\temp\file.txt"
} catch e {
    MsgBox "Error: " e.Message
}
```

## Page 12: Common Pitfalls

- **Using `=` for assignment:**  
  Always use `:=` in V2.
- **Legacy variable expansion (`%var%`):**  
  Not needed in V2, use variables directly.
- **Old command syntax:**  
  Use function and object syntax, not legacy commands.
- **Mixing V1 and V2 code:**  
  V2 is not backward-compatible; always check script version.
```
