## Introduction

**Markdown** is a lightweight markup language that allows you to write formatted text using a plain-text editor.  
It is designed to be easy to read and write, and converts to HTML for web display.  
Markdown is widely used for README files, documentation, blogging, and more.

---

## What is Markdown?

- **Plain text, readable, portable.**
- Converts to clean HTML.
- Supported by GitHub, GitLab, Bitbucket, and many documentation tools.
- Easy to learn: uses simple symbols for formatting.

---

## Basic Syntax

### Headings

Create headings using `#` symbols:

```
# H1 Heading
## H2 Heading
### H3 Heading
#### H4 Heading
##### H5 Heading
###### H6 Heading
```

**Renders as:**

# H1 Heading
## H2 Heading
### H3 Heading
#### H4 Heading
##### H5 Heading
###### H6 Heading

---

### Paragraphs & Line Breaks

- Separate paragraphs with a blank line.
- End a line with two spaces (`  `) for a line break.

```
This is a paragraph.

This is another paragraph.  
This is a line break.
```

---

### Emphasis (Bold & Italic)

- **Bold:** `**text**` or `__text__`
- *Italic:* `*text*` or `_text_`
- ***Bold & Italic:*** `***text***`

```
**Bold**
*Italic*
***Bold and Italic***
```

---

### Blockquotes

Use `>` for blockquotes (nest with `>>`):

```
> This is a blockquote.
>> Nested blockquote.
```

> This is a blockquote.
>> Nested blockquote.

---

### Lists (Ordered & Unordered)

**Unordered:** Use `-`, `*`, or `+`

```
- Item 1
- Item 2
  - Subitem 2.1
  - Subitem 2.2
```

**Ordered:** Use numbers

```
1. First item
2. Second item
   1. Subitem 2.1
   2. Subitem 2.2
```

---

### Code (Inline & Blocks)

**Inline code:** Use backticks `` ` ``

```
Use the `console.log()` function.
```

**Code block:** Use triple backticks (specify language for syntax highlighting):


```
function hello() {
  console.log("Hello, Markdown!");
}
```


---

### Horizontal Rules

Use three or more dashes, asterisks, or underscores:

```
---
***
___
```

## Advanced Syntax

### Task Lists

```
- [x] Write documentation
- [ ] Add more tests
```

- [x] Write documentation
- [ ] Add more tests

---

### Footnotes

Some Markdown processors support footnotes:

```
Here is a footnote reference.: This is the footnote text.
```

---

### Definition Lists

(Some Markdown processors support:)

```
Term 1
:   Definition 1

Term 2
:   Definition 2
```

---

### Strikethrough

```
~~This text is crossed out~~
```

~~This text is crossed out~~

---

### Escaping Characters

Use a backslash `\` to escape special Markdown characters:

```
\*This text is not italicized\*
```

## Best Practices

- **Keep it readable:** Use whitespace and headings.
- **Use code blocks** for code, not inline formatting.
- **Reference sections** with links for easy navigation.
- **Preview your Markdown** in your editor or on GitHub.
- **Stick to standard Markdown** for maximum compatibility.
