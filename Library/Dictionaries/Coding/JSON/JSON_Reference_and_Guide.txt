## Introduction

**JSON** (JavaScript Object Notation) is a lightweight, text-based data format for representing structured data.  
It is widely used for data exchange between servers and web applications, configuration files, and APIs.

---

## What is JSON?

- **Human-readable and machine-readable.**
- Based on key-value pairs and arrays.
- Used in almost all modern programming languages.
- The de facto standard for web APIs and configuration.

---

## Basic Syntax

### Objects

- Defined with curly braces `{}`.
- Contain key-value pairs.
- Keys must be **double-quoted strings**.

```
{
  "name": "Mojula",
  "age": 28,
  "city": "London"
}
```

---

### Arrays

- Defined with square brackets `[]`.
- Contain ordered lists of values.

```
[
  "apple",
  "banana",
  "orange"
]
```

Or as a property:

```
{
  "fruits": ["apple", "banana", "orange"]
}
```

---

### Values

- Can be a string, number, boolean, null, object, or array.

```
{
  "string": "hello",
  "number": 42,
  "float": 3.14,
  "boolean": true,
  "null_value": null,
  "array": [1,2][3],
  "object": { "a": 1 }
}
```

---

### Comments

**JSON does not support comments.**  
All content must be valid JSON-comments will cause parsers to fail.

---

## Best Practices

- **Use double quotes** for all keys and string values.
- **Keep structure clean and consistent:**  
  - Use clear, descriptive key names.
  - Avoid unnecessary nesting.
- **Use native types:**  
  - Numbers, booleans, null, arrays, and objects.
- **Stable keys:**  
  - Avoid changing key names unless necessary.
- **Consistent formatting:**  
  - Indent with 2 or 4 spaces (no tabs).
- **Validate your JSON** before use (see [Tools](#references--tools)).
- **Top-level object:**  
  - Prefer a single top-level object for API responses and config files[1][3][5].

## JSON Schema

**JSON Schema** is a vocabulary for describing the structure and validation rules of JSON data.

### Why Use JSON Schema?

- **Validate** incoming/outgoing data.
- **Document** expected data formats for APIs.
- **Auto-generate** forms, docs, and code.

### Basic JSON Schema Example

```
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "title": "User",
  "type": "object",
  "properties": {
    "name":    { "type": "string" },
    "age":     { "type": "integer" },
    "email":   { "type": "string", "format": "email" }
  },
  "required": ["name", "email"]
}
```

- `"type"` can be `"object"`, `"array"`, `"string"`, `"number"`, `"integer"`, `"boolean"`, or `"null"`.
- `"properties"` describes each field.
- `"required"` lists mandatory fields.
- `"format"` adds extra validation (e.g., `"email"`).

**Learn more:** [JSON Schema Reference][7]

## Usage Examples

### Config File

```
{
  "theme": "dark",
  "autosave": true,
  "refreshInterval": 30
}
```

### API Response

```
{
  "status": "success",
  "data": {
    "id": 123,
    "username": "mojula",
    "roles": ["admin", "editor"]
  }
}
```

### Array of Objects

```
[
  { "id": 1, "name": "Alpha" },
  { "id": 2, "name": "Beta" }
]
```

## Common Pitfalls

- **Single vs. double quotes:**  
  Only double quotes are allowed for keys and string values.
- **Trailing commas:**  
  No trailing commas allowed.
- **No comments:**  
  Remove all comments before parsing.
- **Data types:**  
  JSON has only one number type (no distinction between int and float).
- **Case sensitivity:**  
  Keys are case-sensitive.
